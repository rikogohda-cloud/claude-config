# EC Financial Analysis: 越境EC企業の会計処理方式検出・実質収益分析

## Input

`$ARGUMENTS` を解釈する:

- `<企業名> <Google Drive URL> [対象期間]` → 実行
  - Google Drive URL: フォルダURL (`/folders/<id>`) またはファイルURL
  - 対象期間: 任意。「直近12ヶ月」「2025年度」等。省略時は取得可能な全期間
- `<企業名> <Google Sheets URL> [対象期間]` → スプレッドシート単体の場合も実行可
- 空 or `help` → 使い方を表示して終了

### help時の出力

```
/ec-financial-analysis <企業名> <Drive/Sheets URL> [対象期間]

越境EC企業の財務データから会計処理方式（グロス/ネット計上）を自動検出し、
実質収益構造を分析します。

例:
  /ec-financial-analysis SAZO https://drive.google.com/drive/folders/xxxxx
  /ec-financial-analysis SAZO https://docs.google.com/spreadsheets/d/xxxxx 直近12ヶ月
```

---

## 品質基準（生成時に遵守）

1. 全数値に **データ区分**（【実績/見込/計画/想定】）+ `> 出所: ファイル名 / シート名` 脚注
2. 推測を含む記述に【想定】タグ。ソースで裏取りできない主張に🚩callout
3. データ不足セクションは「データ不足: 〇〇が必要」と明記（空にしない）
4. 同一指標が複数ソースで異なる場合、両方記載 + 差異理由を脚注
5. 金額は百万円単位（`○○M`）で統一。端数は小数第1位まで

---

## Process

### Phase 1: ファイルスキャン + 分類

1. **URL判定**:
   - `docs.google.com/spreadsheets/d/<id>` → スプレッドシートIDを抽出し直接Phase 2へ
   - `drive.google.com/drive/folders/<id>` → フォルダIDを抽出しファイル一覧取得

2. **フォルダの場合**: `python3 ~/.claude/scripts/gdrive_list_folder.py <folder_id> --recursive` でファイル一覧取得

3. **ファイル分類**（優先順）:
   - **PL系**: 月次PL、損益計算書、セグメント別PL → 最優先で抽出
   - **取引系**: GMV明細、決済代行入金明細（GMO等）、セグメント別売上 → テイクレート算出に必須
   - **BS系**: 貸借対照表 → 補助的に参照
   - **CF系**: 入出金明細、口座明細 → 決済フロー分析に使用
   - **その他**: ピッチ資料、事業計画 → セグメント情報・事業構造の補完

4. **Google Sheets の検出**: `.gsheet` ファイルや Sheets URL があれば、スプレッドシートIDを抽出してMCPアクセス対象としてマーク

5. **PDF の検出**: PDFファイルは `python3 ~/.claude/scripts/gdrive_download.py <file_id> /tmp/ec-analysis/<企業名>/<filename>` でダウンロード

### Phase 2: データ抽出

ファイル3個以下は直接処理。4個以上は Agent subagent で並行処理。

**読み取り方法**:
- Google Sheets → MCP `list_sheets` でシート一覧取得 → `get_sheet_data` で各シート読み取り
- PDF → Read ツール（`pages` パラメータで分割、最大20ページ/回）
- Excel/CSV → Bash で `python3` + openpyxl/pandas で読み取り

**抽出項目**（越境EC特化）:

| カテゴリ | 抽出項目 |
|---------|---------|
| セグメント構造 | セグメント名（国→国の方向）、各セグメントの売上・原価・粗利 |
| PL全体 | 売上高合計、売上原価合計、売上総利益、販管費内訳、営業利益、経常利益 |
| GMV/取扱高 | セグメント別GMV（あれば）、決済代行入金額 |
| 為替関連 | 為替差損益、適用レート、外貨建て取引額 |
| 決済代行 | GMO等の入金明細（グロス入金 or ネット入金の区別） |
| 勘定科目 | 売上の勘定科目名（「商品売上」「手数料収入」「受託販売収入」等） |

**読み取り失敗時**: スキップし、読めたファイルのみで続行。最終報告に失敗ファイルを含める。

### Phase 3: 会計処理方式の自動検出（このスキルの核心）

**セグメント単位で** 5つのシグナルを組み合わせて判定する。

#### シグナル定義

| # | シグナル | 算出方法 | ネット示唆 | グロス示唆 |
|---|---------|---------|-----------|-----------|
| 1 | テイクレート | セグメント売上 ÷ GMV | < 20% | > 80% |
| 2 | 売上原価率 | セグメント原価 ÷ セグメント売上 | < 50% | > 85% |
| 3 | セグメント間原価率差 | max(原価率) - min(原価率) | - | 30pt以上の差 |
| 4 | 決済入金/売上比率 | 決済代行入金額 ÷ PL売上 | > 3.0 | 0.8〜1.2 |
| 5 | 勘定科目キーワード | 科目名のテキスト分析 | 「手数料収入」「純額」「受託」 | 「商品売上」「取扱高」「総額」 |

#### スコアリングロジック

各シグナルに -2〜+2 のスコアを付与（ネット方向がマイナス、グロス方向がプラス）:

**シグナル1: テイクレート**
- GMVデータなし → スコア 0（判定不能）
- テイクレート < 10% → -2
- テイクレート 10-20% → -1
- テイクレート 20-80% → 0（判定困難）
- テイクレート 80-95% → +1
- テイクレート > 95% → +2

**シグナル2: 売上原価率**
- 原価データなし → スコア 0
- 原価率 < 30% → -2
- 原価率 30-50% → -1
- 原価率 50-85% → 0
- 原価率 85-95% → +1
- 原価率 > 95% → +2

**シグナル3: セグメント間原価率差**
- セグメント1つのみ → スコア 0
- 差 < 15pt → 0
- 差 15-30pt → +1（混在の可能性）
- 差 > 30pt → +2（グロス/ネット混在の強い示唆）
※ このシグナルはセグメント全体に適用。混在検出時は各セグメントを個別に再評価

**シグナル4: 決済入金/売上比率**
- 決済入金データなし → スコア 0
- 比率 > 5.0 → -2
- 比率 3.0-5.0 → -1
- 比率 1.5-3.0 → 0
- 比率 0.8-1.5 → +1
- 比率 < 0.8 → +2

**シグナル5: 勘定科目キーワード**
- ネットKW検出（手数料収入/純額/受託販売/代理人） → -2
- グロスKW検出（商品売上/取扱高/総額/本人取引） → +2
- 両方 or なし → 0

#### 総合判定

セグメントごとの合計スコア:
- ≤ -3: **ネット計上**【実績】（確信度: 高）
- -2: **ネット計上**【実績】（確信度: 中）
- -1: **ネットの可能性高い**【想定】
- 0: **判定困難** — 🚩要確認
- +1: **グロスの可能性高い**【想定】
- +2: **グロス計上**【実績】（確信度: 中）
- ≥ +3: **グロス計上**【実績】（確信度: 高）

### Phase 4: 実質収益の算出

グロス計上と判定されたセグメントについて実質ネット収益を推定する。

#### テイクレート決定の優先順位

1. **明示データ**: GMVと売上の両方が判明している場合 → `テイクレート = 売上 ÷ GMV`
2. **原価率からの逆算**: 原価率 > 85% の場合 → `推定テイクレート ≈ 1 - 原価率`（粗利≒手数料と仮定）
3. **決済入金からの逆算**: 決済入金 / PL売上 で逆算
4. **業界平均**: 上記すべて不可の場合
   - 越境EC（マーケットプレイス型）: 8-15%
   - 越境EC（転売/仕入型）: 15-30%
   - EC決済代行: 2-5%
   - 🚩業界平均使用時は必ず明記

#### 算出式

```
推定ネット収益 = グロス売上 × 推定テイクレート
グロス部分の見かけ上の利益 = グロス売上 - 推定ネット収益
```

ネット計上セグメントはそのまま実質収益として扱う。

全セグメント合算で:
```
実質総収益 = Σ(ネット計上セグメントの売上) + Σ(グロス計上セグメントの推定ネット収益)
P&L計上額との乖離 = P&L売上合計 - 実質総収益
乖離率 = 乖離額 ÷ P&L売上合計 × 100%
```

## ルール

- AskUserQuestion は使用しない（非対話で完結）
- エラー時も可能な範囲で続行。読めないファイルはスキップし最終報告に含める
- Google Sheets へのアクセスは**必ずMCPツール**を使用（WebFetchは禁止）
- Drive フォルダへのアクセスは `~/.claude/scripts/gdrive_list_folder.py` / `gdrive_download.py` を使用
- 金額は百万円単位（`○○M`）で統一
- セグメント名が不明確な場合、取引方向（例: 日本→韓国）で命名
- テイクレートの業界平均を使用した場合は🚩マークで明示
- 会計処理方式の判定根拠は必ず定量的に示す（「〜と思われる」等の曖昧表現を避ける）
