## 1on1 準備モード

### Step 1: メンバー特定
- $ARGUMENTS からメンバーを特定（上記マッピング参照）
- relationships.md で詳細情報を確認

### Step 2: 情報収集（並行実行）

**2a. 前回の1on1メモ**
- Notion検索:
```
mcp__notion__notion-search({ query: "1on1 <メンバー名>" })
```
- `~/.claude/private/` 配下に1on1メモファイルがあれば読み込み

**2b. 直近のSlackやり取り**
```
mcp__slack__slack_search_messages({ query: "from:<member_name> after:<2週間前>", count: 15 })
mcp__slack-capital__slack_search_messages({ query: "from:<member_name> after:<2週間前>", count: 15 })
```

**2c. 関連タスク**
- todo.md からメンバーに関連するタスクを抽出
- collections.md からメンバー担当の案件を確認（債権回収メンバーの場合）

**2d. 前回のアクションアイテム**
- 前回1on1メモからの未完了アクションアイテム

### Step 3: 準備メモ生成

**重要: 1on1は進捗報告会ではない。「人」にフォーカスすること。**
- 進捗確認はメンバーが作成した定例メモやSlack非同期で十分カバーされる
- 1on1では「状況（What）」ではなく「本人の状態・考え（How/Why）」を掘り下げる
- 「PCI DSS進捗は？」→ NG（進捗確認）。「PCI DSS対応の負荷感どう？」→ OK（本人の状態）

```
## 1on1 準備メモ: [メンバー名]
**日時**: YYYY-MM-DD
**前回**: YYYY-MM-DD（前回のメモがあれば）

### コンディション確認
- [直近の業務負荷の観察（並行案件数、Slack活動量など）]
- [体調・メンタル面で気になる兆候があれば]

### 前回からの変化・気づき
- [前回1on1以降に起きた大きな変化や成果]
- [Slackでの発言から読み取れるトーン・関心の変化]

### 聞きたいこと（人にフォーカス）
1. **負荷・優先順位**: 今一番しんどいこと、落とせるものはないか
2. **マネジメント**: チームメンバーとの関係、困っていること（マネージャーの場合）
3. **成長・キャリア**: 今の仕事で伸びている実感、やりたいことの変化
4. **Naoへのリクエスト**: 自分がブロッカーになっていないか、サポートできること

### メモ欄
（1on1中に記入）
```

### Step 4: 1on1後の記録

AskUserQuestion で:
- 1on1の内容をメモするか確認
- メモがあれば `~/.claude/private/1on1/<member>_YYYYMMDD.md` に保存
- 新しいアクションアイテムがあれば todo.md に追記

## フィードバックモード

### Step 1: メンバー特定
- $ARGUMENTS から "feedback" を除去してメンバーを特定

### Step 2: 情報収集（並行実行）

**2a. 過去の1on1メモ（直近3ヶ月）**
```bash
ls ~/.claude/private/1on1/<member>_*.md
```
- Notion でも検索

**2b. 担当プロジェクトの成果**
- Slackでの活動を検索（広めの期間）
- collections.md の担当案件の進捗（該当する場合）

**2c. 前回のフィードバック**
- 前回のフィードバック文書があれば取得

### Step 3: フィードバック文書ドラフト

```
## フィードバック: [メンバー名]
**期間**: YYYY-MM ～ YYYY-MM
**記入者**: 石神直樹

### 良かった点（Keep）
1. [具体的なエピソード・成果]

### 改善点（Improve）
1. [具体的な改善ポイントと期待]

### 期待・目標（Try）
1. [次期に期待すること]

### 総合コメント
[メンバーへのメッセージ]
```

### Step 4: レビュー

- AskUserQuestion でドラフトの修正を受け付ける
- 修正後、ファイルに保存: `~/.claude/private/feedback/<member>_YYYYMM.md`

## Notes
- フィードバックはフォーマル寄りのトーンで作成
- 具体的なエピソードを必ず含める（抽象的な評価を避ける）
- relationships.md のトーン情報を参考にメンバーとの関係性を把握
- 中間フィードバック（HRBPの滝田さん経由）の文脈も考慮
