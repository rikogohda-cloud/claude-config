# 月次本部長会議 叩き台自動生成

引数: `$ARGUMENTS`（オプション: 対象月 `YYYY-MM`。未指定なら翌月 or 当月末なら当月）

## 重要ルール
- AskUserQuestion は使用しない（非対話で完結）
- 週次データが無いセクションは「（週次データなし）」と明記する
- DB から最新データを取得して反映する
- エラーが発生しても可能な範囲で続行する

---

## Phase 1: 対象月と月次ページの特定

### 1-1. 対象月の決定
- 引数 `$ARGUMENTS` に `YYYY-MM` が指定されていればその月
- 未指定なら、今日の日付を基に判断:
  - 15日以降 → 翌月の会議準備
  - 14日以前 → 当月の会議準備

### 1-2. 月次ページの特定
Notion で「Monthly本部長会議」を検索し、対象月のページを見つける:
- `mcp__notion__notion-search` で `Monthly本部長会議_YYYYMM` を検索
- 親ページ `29393c7ce32d80f4b140efccb98bf7a1`（本部長会議について）配下から特定
- 見つからなければ新規作成する（親ページ配下に）

### 1-3. 既存月次ページの現在コンテンツを取得
- `mcp__notion__notion-fetch` で月次ページの現在の内容を取得（手動入力済みの項目を上書きしないため）

---

## Phase 2: 週次データの収集

### 2-1. 対象となる週次ページの特定
対象月に属する週次ページ（通常4回分）を特定する:
- `mcp__notion__notion-fetch` で親ページ `29393c7ce32d80f4b140efccb98bf7a1` を取得
- タイトルパターン `本部長会議_YYYYMMDD` から対象月のページを抽出
- 日付順にソート（W1, W2, W3, W4）

### 2-2. 週次ページの一括取得
特定した週次ページ（最大5ページ）を **Task subagent を並行起動して** 取得する:

各 subagent への指示:
```
以下のNotionページを fetch して、本部ごとのレポート内容を抽出してください。
ツール: mcp__notion__notion-fetch
ページID: <page_id>

各本部について以下を抽出:
1. 本部名（担当者名）
2. 天気図（☀️/⛅/🌧️/⛈️ など）
3. 3行サマリー（What / Why / So what）
4. 今週の前進（P1-P3）
5. 来週の課題（P1-P3）
6. 議題・全社接続の内容

また、以下のページ全体情報も:
- アジェンダ内容
- 前回アクションテーブルの内容
- 議事録の内容（あれば）

JSON形式で構造化して返してください。
```

## Phase 4: 月次コンテンツの生成

週次データとDBデータを統合して、以下の月次フォーマットに変換する。

### 4-0. 今月の全体サマリー

**天気図テーブル**: 各事業・機能の天気図を W1〜W4 の推移で表示

```
<table>
  <thead>
    <tr><th>事業・機能</th><th>W1</th><th>W2</th><th>W3</th><th>W4</th><th>トレンド</th></tr>
  </thead>
  <tbody>
    <tr><td>UPSIDERカード/PRESIDENTカード</td><td>☀️</td><td>☀️</td><td>⛅</td><td>☀️</td><td>→</td></tr>
    ...（各事業・機能分）
  </tbody>
</table>
```

**Executive Summary**（Nao事前記入用の叩き台）:
- 週次の3行サマリーを月単位で統合
- What: 今月の主要な動き（3-5行）
- Why: その背景・要因
- So what: 経営への示唆・来月への含意

### 4-1. 全社数値（BS/PL/CF）
- このセクションは Tetsu 担当のため **「← Tetsu（事前記入）」** と記載
- フォーマットの枠だけ用意する

### 4-2. 事業別レビュー

各事業・機能について、週次データを集約:

**本部マッピング（Weekly → Monthly）**:

| Weekly 本部 | Monthly セクション | 担当 |
|------------|-------------------|------|
| カード事業本部 | UPSIDERカード/PRESIDENTカード | Kome |
| 支払い.com事業本部 | 支払い.com | Kome |
| UPSIDER AI事業本部 | AI経理 | Kome |
| 融資事業本部 | ベンチャーデット | Nao |
| Card基盤本部 + 経営管理本部（Credit部分） | Credit | Nao/Yusuke |
| 経営管理本部（Finance部分） | Finance/Accounting | Tetsu |
| 共通基盤本部 | IT/Security | Nao |
| Legal&Compliance本部 | Legal/Compliance | Kensuke |
| HR本部 | HR | Toru/Kohei |

各セクションのフォーマット:
```markdown
▶ **セクション名**（担当者）
	### 天気図の推移
	W1(MM/DD): ☀️ → W2(MM/DD): ⛅ → W3(MM/DD): ⛅ → W4(MM/DD): ☀️

	### 月間サマリー
	- What: （週次3行サマリーの集約）
	- Why: （背景の集約）
	- So what: （示唆の集約）

	### 今月の前進 Top 3
	1. （週次の「今週の前進」から最も重要な3つを選定）
	2. ...
	3. ...

	### 来月の課題
	- P1: （最終週の「来週の課題」+ 月を通して解決しなかった課題）
	- P2: ...
	- P3: ...

	### 中長期課題（変更時のみ）
	（週次で言及された中長期的な課題があれば記載）

	### 経営/他部門への依頼
	（週次の「議題・全社接続」から全社レベルの依頼を抽出）
```

### 4-3. 決議事項
- 週次のアクションテーブルや議題から、月次レベルの決議事項候補を抽出
- 「← Nao, Tetsu, Kensuke（事前記入）」と注記

### 4-4. 来月の重点テーマ
- **全社 P1-P3**: 週次の課題から全社レベルのテーマを抽出（叩き台）「← Nao（事前記入）」
- **本部別**: 各本部の最終週の課題を転記「← 各本部長（事前記入）」

## Phase 5: Notionページ更新

`mcp__notion__notion-update-page` で月次ページを更新する。

### 更新ルール
- `replace_content` で全体を置き換える
- 既存の手動入力内容がある場合は、Phase 1-3 で取得した既存コンテンツと照合し:
  - 手動入力済みセクション → そのまま保持
  - 未記入（「未更新」「TBD」「← 担当者名」のみ）→ 自動生成内容で置き換え

### Notion Markdown フォーマット

```markdown
<callout>
💡 このページは週次データ（W1〜W4）から自動生成された叩き台です。各担当は自分のセクションを確認・加筆してください。
自動生成日: YYYY-MM-DD
</callout>

# ⏱ 本日のアジェンダ
（省略 - 月次テンプレートのアジェンダセクションをそのまま使用）

# 📌 前回アクション追跡
（前月の月次ページからアクションを引き継ぎ - 可能であれば）

<table>
  <thead>
    <tr><th>アクション</th><th>担当</th><th>期限</th><th>状況</th><th>備考</th></tr>
  </thead>
  <tbody>
    （週次のアクションテーブルから未完了分を集約）
  </tbody>
</table>

# 0. 今月の全体サマリー
（Phase 4-0 の内容）

# 1. 全社数値（BS/PL/CF）
← Tetsu（事前記入）

# 2. 事業別レビュー
（Phase 4-2 の各セクション）

<database>KPIスクショDB - インライン</database>

# 3. 決議事項
（Phase 4-3 の内容）

# 4. 来月の重点テーマ
（Phase 4-4 の内容）

# 5. 議事録（Gemini）
（空欄 - 会議当日に記入）
```

## Phase 6: 完了報告

最終的に以下を出力する:
1. 更新した月次ページのタイトルとURL（あれば）
2. 対象月と集約した週次ページ数
3. 各セクションの自動生成状況（生成済み / 手動入力保持 / データなし）
4. DB登録件数（リスク・PJ等の月間アクティブ件数）
5. 担当者が事前記入すべき箇所のリスト
