# 本部長会議後 DB登録

引数: `$ARGUMENTS`（オプション: ページID または日付 `YYYYMMDD`。未指定なら最新の本部長会議ページ）

## 重要ルール
- AskUserQuestion は使用しない（非対話で完結）
- 重複登録を避けるため、既存DBエントリと必ず照合する
- 登録前に一覧を出力し、登録結果もすべて報告する
- エラーが発生しても可能な範囲で続行する

---

## Phase 1: 会議ページの取得

### 1-1. 対象ページの特定
- 引数 `$ARGUMENTS` にページIDまたは日付が指定されていれば、それを使用
  - UUID形式 → 直接 `mcp__notion__notion-fetch`
  - `YYYYMMDD` 形式 → 親ページ `29393c7ce32d80f4b140efccb98bf7a1` から `本部長会議_YYYYMMDD` を検索
- 未指定なら、親ページから最新の `本部長会議_YYYYMMDD` を特定

### 1-2. ページ内容の取得
- `mcp__notion__notion-fetch` でページ全体を取得
- 以下のセクションを抽出:
  - 「議題・意思決定」テーブル
  - 「ネクストアクション」テーブル
  - 「議事録 by Gemini」セクション
  - 各本部の「議題・全社接続」セクション
  - 天気図サマリーテーブル

---

## Phase 2: 登録対象の抽出

議事録・意思決定・本部レポートから以下を抽出:

### 2-1. プロジェクト候補
- 新規PJの言及（「新プロジェクト」「PJ立ち上げ」「新施策」等）
- 既存PJのステータス変更（「完了」「延期」「方針変更」等）
- 抽出項目: タイトル、本部、ステータス、今週の差分

### 2-2. リスク・インシデント候補
- 障害報告、セキュリティインシデント
- コンプライアンスリスク
- オペレーションリスク
- 抽出項目: タイトル、本部、影響・背景、対応・ネクストステップ、発生日

### 2-3. 労務インシデント候補
- 退職・休職の問題
- ハラスメント・メンタルヘルス関連
- 採用関連のリスク
- 抽出項目: タイトル、本部、内容、対応状況

### 2-4. 顧客クレーム候補
- 顧客からの苦情・問題報告
- サービス品質に関する問題
- 抽出項目: タイトル、本部、顧客名（あれば）、内容、重大度

---

## Phase 3: 既存データとの重複チェック

4つのDBを並行クエリして、同一案件が既に登録されていないか確認:

- **プロジェクトDB** (`collection://109b79ac-15cf-4c34-b5d9-4596e7b7cdad`)
  - タイトルの類似性で判定（完全一致 or 部分一致）
- **リスク & インシデントDB** (`collection://d1177745-5429-4c1c-bb3a-687c0e5f1dbb`)
  - タイトル + 発生日で判定
- **労務インシデントDB** (`collection://845750c1-2f84-4ffa-80ac-8903298f088d`)
  - タイトルの類似性で判定
- **顧客クレームDB** (`collection://e28b0381-bcba-4645-86b5-44936e651851`)
  - タイトル + 顧客名で判定

ツール: `mcp__notion__notion-query-data-sources`

---

## Phase 4: DB登録

新規エントリのみ登録（重複はスキップ）。

### 4-1. プロジェクトDB
```json
{
  "parentId": "109b79ac-15cf-4c34-b5d9-4596e7b7cdad",
  "title": "プロジェクト名",
  "properties": {
    "ステータス": { "select": "On Track" },
    "フェーズ": { "select": "進行中" },
    "今週の差分": { "text": "進捗内容" },
    "本部": { "select": "本部名" }
  }
}
```

### 4-2. リスク & インシデントDB
```json
{
  "parentId": "d1177745-5429-4c1c-bb3a-687c0e5f1dbb",
  "title": "インシデント内容",
  "properties": {
    "本部": { "multi_select": "[\"本部名\"]" },
    "状況": { "select": "Open" },
    "影響・背景": { "text": "概要" },
    "対応・ネクストステップ": { "text": "対応内容" },
    "発生日": { "date": "YYYY-MM-DD" }
  }
}
```

### 4-3. 労務インシデントDB
```json
{
  "parentId": "845750c1-2f84-4ffa-80ac-8903298f088d",
  "title": "インシデント内容",
  "properties": {
    "本部": { "select": "本部名" },
    "対応状況": { "select": "対応中" }
  }
}
```

### 4-4. 顧客クレームDB
```json
{
  "parentId": "e28b0381-bcba-4645-86b5-44936e651851",
  "title": "クレーム内容",
  "properties": {
    "本部": { "select": "本部名" },
    "重大度": { "select": "High" },
    "対応状況": { "select": "対応中" }
  }
}
```

ツール: `mcp__notion__notion-create-pages`

---

## Phase 5: 完了報告

最終的に以下を出力する:
1. **登録件数サマリー**
   - プロジェクトDB: N件（新規）/ M件（スキップ: 重複）
   - リスク & インシデントDB: N件 / M件
   - 労務インシデントDB: N件 / M件
   - 顧客クレームDB: N件 / M件
2. **登録したエントリの一覧**（タイトル、DB名、本部）
3. **スキップした重複エントリ**（あれば、理由付き）
4. 特筆すべきエラーや警告（あれば）
